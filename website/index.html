WAF & SOC Engine — Deep Dive

A production-grade security pipeline processing every HTTP request through multiple analysis layers. 26 signatures, 7 behavioral rules, honeypot detection, IP reputation, and multi-phase attack correlation.

1

Traffic Ingestion

Nginx reverse proxy captures every HTTP/HTTPS request with rate limiting (general: 10r/s, login: 2r/m, API: 30r/s). A log processor tails access logs in real-time and streams structured JSON events to Apache Kafka.

2

26 WAF Signatures + 7 Behavioral Rules

Injection (13): SQLi, XSS, Command Injection, XXE, NoSQL, SSTI, CRLF, Log4Shell, LDAP. Access Control (3): Path Traversal/LFI, Scanner Detection, WebSocket Hijack. Plus: SSRF, Brute Force, HTTP Smuggling, Deserialization, Cryptominer, API Abuse, Exposed Configs. Behavioral: Recon, Exploit Attempts, Malformed Requests, Brute Force, Slow Attacks, Data Exfiltration.

3

Honeypot & IP Reputation

Fake endpoints (/wp-admin, /.env, /phpmyadmin) instantly identify attackers (WAF-5001, 0.99 confidence). Persistent IP reputation scoring in PostgreSQL tracks attack history per IP. GeoIP via DB-IP Lite for geographic threat intelligence.

4

Multi-Phase Attack Correlation

Detects attack chains: Recon → Probe → Exploit → Exfiltration. Groups same-IP attacks within 600s windows into episodes. Severity auto-escalates based on event count, types, and attack phases. OWASP Top 10 2021 classification for every threat.

5

Instant Blocking & Auto-Response

Tier 1: Instant block on critical signatures (confidence ≥ 0.90) — first hit, no waiting.
Tier 2: Episode-based block after 3+ attack episodes.
Tier 3: Rate-limit block at 40 req/5min threshold.
Nginx blocklist updated with zero-downtime reload via sidecar container. 24h default block duration.

6

Supreme SOC Dashboard

Real-time SSE attack feed, world attack map, OWASP breakdown, kill chain visualization, attack heatmap (hour × day), AbuseIPDB threat intelligence, signature monitor with hit counts, IP reputation tracking, CSV export, and cyberpunk dark theme. JWT authentication with full audit trail in PostgreSQL.

3-Layer Security Architecture

Security isn't a feature — it's the foundation. Three independent layers protect every interaction from prompt to execution.

🚨

Prompt Guard

WAF for LLM prompts. Detects injection attacks BEFORE reaching the model: role override attempts, system prompt extraction, delimiter injection, encoding attacks (base64/hex), and tool abuse patterns. Full audit logging.

🧹

Input Sanitizer

Blocks dangerous commands BEFORE tool execution: reverse shells (nc -e, bash -i), crypto miners, fork bombs, destructive commands (rm -rf /, mkfs, dd), SQL injection, and path traversal attempts.

🔒

Secure Channel

API key authentication (Bearer token / X-API-Key), rate limiting per client (60 req/min), Telegram ACL with owner-only admin commands. JWT for dashboard. Zero hardcoded secrets — everything from environment variables.

✅

Action Confirmations

Dangerous actions require explicit confirmation. Conservative defaults prevent accidental destruction.

🔐

No Secrets in Code

All secrets from .env. Comprehensive .gitignore. Zero hardcoded credentials.

🛡️

Shell Control

HOST_CONTROL_ALLOW_RUN=false by default. SSH key-based auth only. All remote commands require confirmation.

Quickstart — Up in 3 Steps

Works on any Linux machine with Docker. Raspberry Pi, laptop, server, or cloud VM.

1

Clone & Configure

git clone https://github.com/TokioAI/TokioAI.git
cd TokioAI
cp .env.example .env

# Edit .env with your API keys:
# ANTHROPIC_API_KEY or OPENAI_API_KEY
# POSTGRES_PASSWORD
# TELEGRAM_BOT_TOKEN (optional)
nano .env
2

Start the Stack

docker compose up -d --build

# Verify everything is running
docker compose ps

# Or use the setup wizard
python3 -m tokio_agent.setup_wizard
3

Use TokioAI

# Via REST API
curl -X POST http://localhost:8000/chat \
  -H "Authorization: Bearer YOUR_KEY" \
  -d '{"message": "Deploy mysite.com on GCP"}'

# Via Telegram — just talk to your bot
# Via CLI
python3 -m tokio_agent.cli

3 Deployment Modes

Full Local: Everything on your machine (dev, testing, Raspberry Pi).
Hybrid: Agent local, WAF/Kafka/DB in cloud.
Full Cloud: Everything on GCP. Maximum availability.
